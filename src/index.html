Gestion Stock Pharmacie

Réalisation d'une application de type "client lourd" faite en Java. Cette application prend place dans un contexte hospitalier. Il a fallu corriger une application non fonctionnelle et s'assurer de la protection des données des utilisateurs. Gestion Rendez-vous

Ce projet prend place dans le même contexte hospitalier que précedemment. Il s'agit cette fois d'une application dite "client léger". Cette application réalisée en Symfony et permet la gestion des rendez-vous pour des médecins. Plusieurs rôles interviennent : les patients qui passent des demandes de rendez-vous, les assistants qui valident ou non les rendez-vous selon les disponibilités du médecin. Les médecins qui peuvent consulter les rendez-vous à une date donnée.

Gestion Rendez-vous Mobile

Ce projet est un portage de l'application Symfony sur Flutter. Elle est également de type "client lourd" mais est disponible sur mobile. Cette application est encore en cours de développement. Elle commence sur l'application Symfony avec l'installation d'une API afin de réaliser des requêtes HTTP pour retourner des objets JSON. Une fois ces objets reçus, ils sont traités par le programme Flutter et affiche les informations. L'application demande aux patients de se connecter, une fois connecté, l'application affiche les rendez-vous de l'utilisateur.
